HTTP - (HyperText Transfer Protocol)

  • Defines what actions web browsers/sites should take in response to command

  • Stateless protocol
    • Doesn't remember last command
    • Therefore, each command is independent

  • Connectionless protocol
    • When client sends request to server, server disconnects from client
    • When server has the necessary info, it connects back to client

Domain Name System (DNS)

  • Allows for obtainment of IP address for certain domain

  • Process

    1. Type in domain (ex: https://google.com.)

    2. If IP address not stored in cache, browser goes to IPS

    3. If IPS doesn't have the domain-IP association stored, browser goes to Root Name Server

    4. Root Name Server refers browser to specific TLD
      • TLD groups are based on .com, .org., etc

    5. TLD refers to a certain Authoritative Name Server
      • These servers contain DNS records, one of which pairs the entered Domain to the needed IP address
      • Broswer requests and obtains this address from ANS

Micro Services

  • way of breaking applications into many smaller applications that can be run on different hardware/server instances

  • All work together to provide functionality of entire application

  • Advantages
    • Don't have to export entire application to check only one "micro" application's functionality
    • If a specific "micro" application's service is needed more than the others, only that one can be made more available
    • Can use different language for each "micro" application

  • Monolithic Architecture
    • Have only one application
    • Disadvantages
      • Difficult to deploy big applications
      • When only one part of application is needed more, availability of entire application has to be increased
        • Instances of lots of unneccessary apps therefore increased

API's : Application Programming Interface

  • Basic Definition: Allows for communication with a software component
  • Application
    • Allows for developers to increase functionality of their applications
    • Instead of having to code in the functionality, can just access it from other source using API's
